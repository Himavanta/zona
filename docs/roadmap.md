# Zona 实现路线图

1. 词法器 ✅
2. 数据栈 + 字面量压栈 ✅
3. 算术和比较运算符 ✅
4. `.` 打印 ✅
5. 栈操作原语（`:dup` `:drop` `:swap` `:over` `:rot`）✅
6. 字定义（`@` `;`）和调用 ✅
7. 控制流（`?` `!` `~` `$`）✅
8. 内存（`&` `#` `:here` `:allot`）✅
9. 字符串 ✅
10. REPL 交互模式 ✅
11. 文件执行模式 ✅

## 待实现（易 → 难）

### 工具链
1. ⭐ 测试框架 — 用 zona 写断言字，自动化测试
2. ⭐ 命令行参数 — 程序能接收外部参数
3. ⭐⭐ REPL 历史记录 — readline 库，上下键翻历史

### 语言特性
4. ⭐ 文件夹即模块 — `:use` 基础上加目录遍历
5. ⭐⭐ 文件读写 — `:fopen` `:fread` `:fwrite` `:fclose`
6. ⭐⭐ 字符串模板 — `'hello $name'`，词法器处理拼接
7. ⭐⭐ 匿名代码块 — 用 `[ ]` 等待选符号实现，提升 `?` 的表达力
8. ⭐⭐ 错误处理机制 — 用户可捕获的错误
9. ⭐⭐ 尾递归优化 — 防止深递归爆返回栈
10. ⭐⭐ 待选符号用途 — `( ) [ ] { } , | " `` ` 按需分配
11. ⭐⭐⭐ 数字类型系统 — 后缀精度如 `3u8`，需改栈存储结构
12. ⭐⭐⭐ 栈类型系统 — 每个元素带类型标记，类型判断原语的前提
13. ⭐⭐⭐ FFI — 调用外部 C 库函数（dlopen/dlsym）
14. ⭐⭐⭐ QBE 编译后端 — 第二阶段，前端复用，生成 IR

## 设计备忘

- 宿主语言：C
- 显式路径前缀（`./` `../`）为未来标准库/包管理路径保留空间
- 文件夹即模块（Hare 风格）：等实际需要时再实现
- 无前缀路径保留给标准库/包管理
- 字可覆盖重定义，后定义遮盖先定义
