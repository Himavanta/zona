# Zona 实现路线图

## 第一阶段：解释器 / REPL ✅

1. 词法器 ✅
2. 数据栈 + 字面量压栈 ✅
3. 算术和比较运算符 ✅
4. `.` 打印 ✅
5. 栈操作原语（`:dup` `:drop` `:swap` `:over` `:rot`）✅
6. 字定义（`@` `;`）和调用 ✅
7. 控制流（`?` `!` `~` `$`）✅
8. 内存（`&` `#` `:here` `:allot`）✅
9. 字符串 ✅
10. REPL 交互模式 ✅
11. 文件执行模式 ✅
12. `:type` `:emit` `:key` IO 原语 ✅
13. `:stack` 调试 ✅
14. `:use` 文件引入 ✅
15. 标准库 `std/` ✅
16. `:alloc` `:free` 动态内存 ✅
17. `:time` `:rand` `:exit` 系统原语 ✅
18. `:argc` `:argv` 命令行参数 ✅
19. `:fopen` `:fread` `:fwrite` `:fclose` 文件读写 ✅
20. VS Code 语法高亮 ✅
21. 错误信息加行号 ✅
22. 测试框架 `std/test.zona` ✅
23. REPL 历史记录（readline）✅

## 待实现（易 → 难）

1. ⭐ 文件夹即模块 — `:use` 基础上加目录遍历
2. ⭐⭐ 字符串模板 — `'hello $name'`，词法器处理拼接
3. ⭐⭐ 匿名代码块 — 用 `[ ]` 等待选符号实现，提升 `?` 的表达力
4. ⭐⭐ 错误处理机制 — 用户可捕获的错误
5. ⭐⭐ 待选符号用途 — `( ) [ ] { } , | " `` ` 按需分配
6. ⭐⭐⭐ 数字类型系统 — 后缀精度如 `3u8`，需改栈存储结构
7. ⭐⭐⭐ 栈类型系统 — 每个元素带类型标记，类型判断原语的前提
8. ⭐⭐⭐ FFI — 调用外部 C 库函数（dlopen/dlsym）
9. ⭐⭐⭐ QBE 编译后端 — 第二阶段，前端复用，生成 IR

## 已讨论不需要的

- 尾递归优化 — `~` 就是循环，不需要
- 块注释 — `_` 行注释够用
- 列号报错 — 行号已足够
- 具名导入导出 — 字典全局扁平
- HTTP / 异步 — 复杂度过高

## 第二阶段：QBE 编译后端

规范稳定后，复用第一阶段的前端，将"解释执行"替换为"生成 QBE IR"。

## 设计备忘

- 宿主语言：C
- 编译命令：`cc zona.c -o zona -lm -lreadline`
- 显式路径前缀（`./` `../`）为未来标准库/包管理路径保留空间
- `[ ]` `( )` 是最有价值的待选符号，预留给匿名代码块或错误处理
- 字可覆盖重定义，后定义遮盖先定义
