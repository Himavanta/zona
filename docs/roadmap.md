# Zona 实现路线图

## 第一阶段：解释器 / REPL ✅

1. 词法器 ✅
2. 数据栈 + 字面量压栈 ✅
3. 算术和比较运算符 ✅
4. `.` 打印 ✅
5. 栈操作原语（`:dup` `:drop` `:swap` `:over` `:rot`）✅
6. 字定义（`@` `;`）和调用 ✅
7. 控制流（`?` `!` `~` `$`）✅
8. 内存（`&` `#` `:here` `:allot`）✅
9. 字符串 ✅
10. REPL 交互模式 ✅
11. 文件执行模式 ✅
12. `:type` 字符串打印 ✅
13. `:emit` 字符输出 ✅
14. `:stack` 调试用栈查看 ✅
15. 文件引入（`:use`）✅
16. 标准库（`std.zona`）✅
17. 动态内存（`:alloc` `:free`）✅

## 待实现（易 → 难）

1. ⭐ 文件夹即模块 — `:use` 基础上加目录遍历
2. ⭐⭐ 字符串模板 — `'hello $name'`，词法器处理拼接
3. ⭐⭐ 错误处理机制 — 用户可捕获的错误，需要设计语法
4. ⭐⭐ 待选符号用途 — `( ) [ ] { } , | " `` ` 按需分配
5. ⭐⭐⭐ 数字类型系统 — 后缀精度如 `3u8`，需改栈存储结构
6. ⭐⭐⭐ 栈类型系统 — 每个元素带类型标记，所有运算加类型检查
7. ⭐⭐⭐ QBE 编译后端 — 第二阶段，前端复用，生成 IR
8. ⭐⭐⭐ FFI — 调用外部 C 库函数（dlopen/dlsym），不用改解释器就能扩展能力

## 设计备忘

- 宿主语言：C
- 显式路径前缀（`./` `../`）为未来标准库/包管理路径保留空间
- 文件夹即模块（Hare 风格）：等实际需要时再实现
- 无前缀路径保留给标准库/包管理
- 字可覆盖重定义，后定义遮盖先定义
