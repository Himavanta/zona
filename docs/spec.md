# Zona 语言规范

## 一、词法规则

- 空白分隔词元
- 全部使用单符号，没有组合复合符号语法

### 1.1 字的命名

- 只能使用字母和数字，第一位必须是字母，不允许下划线
- 用户字、单符号、系统原语三者在词法层面天然不会冲突，因此没有关键字也没有保留字

### 1.2 字面量

字面量在运行时将常量压栈。

#### 数字

- 无特殊符号，自动处理
- 整数、浮点数、负数：`123` `1.3` `-5`
- 数字类型暂不细致讨论，后续可能通过后缀区分精度和类型，如 `3u8`
- `-` 紧贴数字且前面是空白时为负数字面量，否则为减法运算符

#### 布尔值

- `0` 为假，`1` 为真

#### 字符串

- 单引号包裹：`'hello'`
- 转义字符 `\`：`'\n'` `'hello \'tom\''`
- 字符串模板 `$`：暂不实现
- 在运行时压入字符串地址和长度

### 1.3 注释

- 行注释：`_`（从 `_` 到行尾）
- 没有块注释

### 1.4 解析优先级

- 字符串定界符 `'` 优先级高于一切，字符串内部不解析 `_` 等符号

## 二、单符号语法标记

固定，不可重定义。

| 符号 | 功能 |
|------|------|
| `@` | 开始定义字 |
| `;` | 结束定义字（隐式返回） |
| `$` | 显式中途返回 |
| `?` | 条件执行 |
| `!` | else 分支 |
| `~` | 跳回当前字头部 |
| `&` | 内存读 `( addr -- value )` |
| `#` | 内存写 `( value addr -- )` |
| `+` | 加 |
| `-` | 减 |
| `*` | 乘 |
| `/` | 除 |
| `%` | 取模 |
| `^` | 幂 |
| `>` | 大于 |
| `<` | 小于 |
| `=` | 等于 |
| `.` | 打印栈顶值并消耗 `( x -- )` |
| `_` | 行注释 |
| `'` | 字符串定界符 |

没有 `+=` `-=` `*=` `/=` `++` `--` 等复合运算符。
没有 `>=` `<=` `!=` `==` 等复合比较符。

### 待选符号（尚未分配用途）

```
( ) [ ] { } , | " `
```

注意：`[ ]` 和 `( )` 是最有价值的符号对，可能用于匿名代码块或错误处理（try-catch），不应轻易分配给其他用途。

## 三、系统原语

以 `:` 开头，后跟小写字母，作为一个完整词元。用于区分内部原语和用户定义字。

### 3.1 栈操作

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:dup` | 复制栈顶 | `( x -- x x )` |
| `:drop` | 丢弃栈顶 | `( x -- )` |
| `:swap` | 交换栈顶两个 | `( x y -- y x )` |
| `:over` | 复制第二个到栈顶 | `( x y -- x y x )` |
| `:rot` | 旋转前三个 | `( x y z -- y z x )` |

### 3.2 内存

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:here` | 返回当前空闲地址 | `( -- addr )` |
| `:allot` | 分配 n 个单元（线性） | `( n -- )` |
| `:alloc` | 分配 n 个单元（动态） | `( n -- addr )` |
| `:free` | 释放动态分配的地址 | `( addr -- )` |

### 3.3 IO

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:type` | 打印字符串 | `( addr len -- )` |
| `:emit` | 输出单个字符 | `( c -- )` |
| `:key` | 读取一个字符 | `( -- c )` |
| `:fopen` | 打开文件 | `( path_addr path_len mode_addr mode_len -- handle )` |
| `:fread` | 读一个字符 | `( handle -- c )` EOF 时返回 -1 |
| `:fwrite` | 写一个字符 | `( c handle -- )` |
| `:fclose` | 关闭文件 | `( handle -- )` |

### 3.4 系统

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:time` | 当前时间戳 | `( -- n )` |
| `:rand` | 随机数 | `( max -- n )` |
| `:exit` | 退出程序 | `( code -- )` |
| `:argc` | 命令行参数个数 | `( -- n )` |
| `:argv` | 取第 n 个参数为字符串 | `( n -- addr len )` |
| `:use` | 引入文件 | 编译时处理，见第八章 |

### 3.5 调试

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:stack` | 打印栈内所有内容（不消耗） | `( -- )` |

## 四、字定义

```
@ 字名 字定义体 ;
```

- `@` 开始定义，后跟字名
- `;` 结束定义，隐式返回
- `$` 可在定义体中显式提前返回
- 同名字可以重新定义，后定义的遮盖先定义的，已编译的旧引用不受影响

## 五、控制流

### 5.1 条件执行

`?` 和 `!` 只读取接下来的**一个词元**（用户字、系统原语、单符号、数字字面量、字符串字面量均可）。需要执行多条语句时，必须将代码块定义成字来调用。

单分支：

```
条件 ? 字
```

- 消耗栈顶标志，非 0 则执行该字，否则跳过

双分支：

```
条件 ? 真字 ! 假字
```

- 非 0 执行真字，0 执行假字

### 5.2 循环

`~` 无条件跳回当前字头部继续执行。

需要显式使用条件判断来控制循环退出，这是有意设计。

模式一：条件继续

```
@ example
    _ 循环体
    继续条件 ? ~
;
```

模式二：条件退出

```
@ example
    退出条件 ? $
    _ 循环体
    ~
;
```

模式三：条件退出带 else

```
@ example
    _ 循环体
    退出条件 ? 123 ! ~
;
```

## 六、内存

### 6.1 基本单位

以 cell（单元）为基本单位，不暴露字节概念。一个 cell 存储一个值（数字、地址等），底层宽度由实现决定，用户不感知。

### 6.2 读写

- `&` 读一个单元 `( addr -- value )`
- `#` 写一个单元 `( value addr -- )`

### 6.3 分配

线性分配（只增不减）：

- `:here` 返回当前空闲地址 `( -- addr )`
- `:allot` 分配 n 个单元，移动 `:here` 指针 `( n -- )`

动态分配（可释放）：

- `:alloc` 分配 n 个单元，返回地址 `( n -- addr )`
- `:free` 释放该地址 `( addr -- )`

两者操作不同内存区域，互不冲突。`&` `#` 对两种地址都有效。

## 七、程序入口

- 文件执行：`zona file.zona`，从头到尾顺序执行，顶层代码直接运行，定义字和执行代码可以混在一起
- REPL：`zona`（无参数），交互式逐行执行

两者本质相同：读取词元、解释执行。REPL 只是输入源从文件换成 stdin。

## 八、文件引入

使用 `:use` 原语引入其他文件的定义。

```
:use './math.zona'
:use '../lib/utils.zona'
```

规则：
- 路径必须以 `./` 或 `../` 开头，不允许省略前缀
- `:use` 必须出现在文件内所有字定义和执行代码之前
- 同一文件只会被引入一次，重复引入自动跳过（避免循环引入）
- 路径相对于当前文件所在目录

无前缀路径保留给未来可能的标准库或包管理使用。

## 九、待定事项与设计讨论记录

### 已讨论，暂不实现

- **块注释 `( )`**：不需要，`_` 行注释够用，`( )` 留作更有价值的用途
- **字符串模板**：`$` 从栈取值拼接，但词法阶段无法获取运行时栈值，实现复杂。现有 `:type` 和 `.` 拼接能满足需求
- **错误捕获机制**：zona 的 `?` 天然支持"先判断后执行"，大部分错误可预防。等实际遇到需要 try-catch 的场景再设计
- **类型判断原语**：需要栈类型系统支持，当前栈上全是 `double`，无法区分数字和字符串地址
- **HTTP 请求 / 异步**：复杂度过高，与 zona 精简定位不符
- **具名导入导出**：不需要，字典全局扁平，`:use` 执行文件即引入所有字
- **列号报错**：zona token 很短，行号已足够定位问题
- **尾递归优化**：不需要，zona 的 `~` 就是循环，不消耗返回栈，深递归应改写为 `~` 循环

### 待实现

- **数字类型系统**：后缀精度如 `3u8`，需改栈存储结构为 tagged union，影响面大
- **栈类型系统**：每个元素带类型标记，所有运算加类型检查，是类型判断原语的前提
- **文件夹即模块**：`:use './math'` 引入目录下所有 `.zona` 文件，`:use` 基础上加目录遍历
- **FFI**：通过 `dlopen`/`dlsym` 调用外部 C 库函数，不改解释器即可扩展能力
- **QBE 编译后端**：复用前端，生成 QBE IR，编译为原生代码
- **待选符号**：`( ) [ ] { } , | " `` ` 用途待定，按需分配

### 设计原则

- 数字当前统一使用 `double`（IEEE 754），与 JS 相同，存在 `0.1 + 0.2 != 0.3` 问题，精度 15-16 位有效数字，现阶段够用
- 栈为无类型栈（传统 Forth 风格），数字和字符串地址混在栈上
- 字可覆盖重定义，方便 REPL 调试和标准库定制
- `:use` 路径强制 `./` `../` 前缀，无前缀保留给未来标准库/包管理
- 核心保持精简，复杂功能通过标准库（zona 实现）和 FFI（C 库）扩展
