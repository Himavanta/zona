# 最终语言规范（严格依您的定义）

## 一、词法规则

- **空白**分隔词元。
- **注释**：`_` 直到行尾，无块注释。
- **数字字面量**：整数（如 `123`、`-5`）或浮点数（如 `1.3`）。负号必须紧贴数字。
- **字符串字面量**：单引号 `'` 包围，支持转义 `\'` `\n` `\t` `\\`。例如：`'hello'`。不实现模板。
- **单符号语法标记**（固定，不可重定义）：
  ```
  @ ; ? ! ~ $ & # + - * / % ^ > < = _ '
  ```
- **系统原语字**：以 `:` 开头，后跟一个或多个**小写字母**。例如 `:dup`、`:here`。整个作为一个词元。
- **用户定义字名称**：以字母开头，后跟零个或多个字母或数字。不能与任何系统原语字或单符号语法标记重名。

## 二、内核原语完整列表

### 2.1 单符号语法标记

| 符号 | 功能 | 约束 |
|------|------|------|
| `@` | 开始定义字 | 后跟用户字名，直到 `;` |
| `;` | 结束定义字 | 隐式返回 |
| `?` | 条件执行 | 后面**必须且只能跟一个字**（或一个数字字面量/字符串字面量） |
| `!` | else 分隔符 | 与 `?` 配对：`? 真字 ! 假字`，真字和假字各自只能是一个字 |
| `~` | 跳回当前字开头 | 无条件 |
| `$` | 立即从当前字返回 | 无 |
| `&` | 内存读 | `( addr -- value )` |
| `#` | 内存写 | `( value addr -- )` |
| `+ - * / % ^` | 算术运算 | 二元运算，结果压栈 |
| `> < =` | 比较运算 | 结果压栈，`0` 或 `1` |
| `_` | 行注释 | 忽略到行尾 |
| `'` | 字符串定界符 | 包裹字符串字面量 |

### 2.2 系统原语字（`:xxx` 形式）

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:dup` | 复制栈顶 | `( x -- x x )` |
| `:drop` | 丢弃栈顶 | `( x -- )` |
| `:swap` | 交换栈顶两个 | `( x y -- y x )` |
| `:over` | 复制第二个 | `( x y -- x y x )` |
| `:rot` | 旋转三个 | `( x y z -- y z x )` |
| `:here` | 返回字典指针 | `( -- addr )` |
| `:allot` | 分配 n 字节 | `( n -- )` |
| `:emit` | 输出字符 | `( c -- )` |
| `:execute` | 执行地址 | `( xt -- )` |

> 没有 `:exit`（使用 `$` 即可），没有 `>r`、`r>`、`0=`、`1-` 等。这些都可以用 `:xxx` 或用户字实现。

## 三、字定义语法

```
@ 字名
    字定义体
;
```

- **字名**：用户定义字名称（字母开头，字母数字）。
- **字定义体**：零个或多个词元（数字、字符串、单符号、系统原语、用户字）。
- `;` 结束定义，并隐式生成返回。也可在定义体中使用 `$` 提前返回。

## 四、控制流详细规则

### 4.1 条件执行（单分支）
```
? 字
```
- 栈顶必须有一个标志（0 或非 0）。`?` 消耗该标志。
- 如果标志非 0，则执行后面的**那一个字**（只能是单个字，不能是一串代码）。
- 如果标志为 0，则跳过该字。

### 4.2 条件执行（双分支）
```
? 真字 ! 假字
```
- 栈顶标志非 0 时执行 `真字`，否则执行 `假字`。`真字` 和 `假字` 各自**必须是一个字**。
- `!` 是分隔符，不是独立操作。

### 4.3 循环
- 只有 `~` 一个符号，它无条件跳转到当前字的第一条指令（即字体的开始）。
- 要退出循环，必须在循环体内的某个位置**不执行** `~`。通常做法是使用 `?` 条件跳过 `~`。例如：
```
@ loop-example
    循环体代码
    继续条件 ? ~   _ 如果继续条件为真，则 ~ 被跳过，字自然结束（退出循环）；否则执行 ~ 跳回开头
;
```
或者使用提前返回 `$`：
```
@ loop-example
    退出条件 ? $
    循环体代码
    ~
;
```

## 五、内存管理

- 使用 `:here` 获取当前字典空闲地址。
- 使用 `:allot` 分配字节（移动 `:here` 指针）。
- 使用 `&` 和 `#` 读写地址。

示例：定义一个变量 `counter`
```
@ counter
    :here :dup :allot 1   _ 分配 1 个存储单元，地址留在栈上，然后被 counter 记住？
```
更好的方式：`counter` 是一个字，执行时压入变量的地址。传统上 `variable` 的实现需要 `create` 和 `does>`，但这里我们简化。用户可以这样：
```
@ counter-addr
    :here   _ 返回当前地址
    :allot 1
;
```
然后使用：
```
counter-addr &    _ 读
123 counter-addr #   _ 写
```

## 六、示例代码（全部使用 `_` 注释，不引入任何额外符号）

### 示例 1：阶乘（递归）
```
@ factorial
    dup 1 <= ? :drop 1 !   _ 若 n<=1，丢弃 n 并压入 1
    dup 1 - factorial *
;

5 factorial   _ 假设标准库中有打印数字的字，例如 :print
```

### 示例 2：从 n 倒数到 1 并打印
```
@ dec  1 - ;               _ 定义递减字
@ countdown
    dup 0= ? $             _ 若为0返回
    dup :print             _ 打印（:print 是标准库字）
    dec                    _ 减一
    ~
;
10 countdown
```

### 示例 3：使用内存变量累加
```
@ counter   _ 分配一个变量，执行时压入地址
    :here :dup :allot 1    _ 留一个单元，地址在栈上，但这里如何返回？需要调整
```
因为 `@` 定义的字体最后压栈的值会成为该字的返回值。所以上面定义中，`:here` 压入当前地址，`:dup` 复制它，`:allot 1` 消耗一个地址推进指针，另一个地址留在栈上作为 `counter` 执行时压入的值。正确写法：
```
@ counter   _ 执行时 counter 返回变量的地址
    :here       _ 压入当前空闲地址
    :dup        _ 保留一份给 :allot
    :allot 1    _ 分配 1 单元
;               _ 栈上还剩最初的地址，作为返回值
```

使用：
```
0 counter #               _ 写入 0
counter & :print          _ 读取并打印
```

## 七、明确禁止的写法

- 任何名字中包含符号（如 `1+`、`0=`、`>r`、`r>`）。
- 使用 `\` 作为注释（必须用 `_`）。
- 使用未定义的单符号（如 `,` `.` `:` 单独出现等）。
- 在 `?` 或 `? !` 后面跟多个字或代码块。
- 用户字与系统原语或单符号重名。
