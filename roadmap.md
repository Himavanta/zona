# Zona 实现路线图

## 第一阶段：解释器 / REPL ✅

1. 词法器（tokenizer）✅
2. 数据栈 + 字面量压栈 ✅
3. 算术和比较运算符 ✅
4. `.` 打印 ✅
5. 栈操作原语（`:dup` `:drop` `:swap` `:over` `:rot`）✅
6. 字定义（`@` `;`）和调用 ✅
7. 控制流（`?` `!` `~` `$`）✅
8. 内存（`&` `#` `:here` `:allot`）✅
9. 字符串 ✅
10. REPL 交互模式 ✅
11. 文件执行模式 ✅

## 第一阶段补充

12. `:type` 字符串打印 ✅
13. `:emit` 字符输出 ✅
14. `:stack` 调试用栈查看 ✅
15. 文件引入（`:use`）
16. 标准库（`std.zona`）

## 第二阶段：QBE 编译后端

规范稳定后，复用第一阶段的前端（词法、解析），将"解释执行"替换为"生成 QBE IR"。

- QBE 负责寄存器分配、指令选择、优化，输出汇编
- 最终通过系统汇编器和链接器生成可执行文件
- 解释器和编译器共存，REPL 始终可用

## 设计备忘

- 宿主语言：C
- 显式路径前缀（`./` `../`）为未来标准库/包管理路径保留空间
- 文件夹即模块（Hare 风格）：等实际需要时再实现
- VS Code 语法高亮：优先级高，投入小回报大
