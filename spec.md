# Zona 语言规范

## 一、词法规则

- 空白分隔词元
- 全部使用单符号，没有组合复合符号语法

### 1.1 字的命名

- 只能使用字母和数字，第一位必须是字母，不允许下划线
- 用户字、单符号、系统原语三者在词法层面天然不会冲突，因此没有关键字也没有保留字

### 1.2 字面量

字面量在运行时将常量压栈。

#### 数字

- 无特殊符号，自动处理
- 整数、浮点数、负数：`123` `1.3` `-5`
- 数字类型暂不细致讨论，后续可能通过后缀区分精度和类型，如 `3u8`
- `-` 紧贴数字且前面是空白时为负数字面量，否则为减法运算符

#### 布尔值

- `0` 为假，`1` 为真

#### 字符串

- 单引号包裹：`'hello'`
- 转义字符 `\`：`'\n'` `'hello \'tom\''`
- 字符串模板 `$`：暂不实现
- 在运行时压入字符串地址和长度

### 1.3 注释

- 行注释：`_`（从 `_` 到行尾）
- 没有块注释

### 1.4 解析优先级

- 字符串定界符 `'` 优先级高于一切，字符串内部不解析 `_` 等符号

## 二、单符号语法标记

固定，不可重定义。

| 符号 | 功能 |
|------|------|
| `@` | 开始定义字 |
| `;` | 结束定义字（隐式返回） |
| `$` | 显式中途返回 |
| `?` | 条件执行 |
| `!` | else 分支 |
| `~` | 跳回当前字头部 |
| `&` | 内存读 `( addr -- value )` |
| `#` | 内存写 `( value addr -- )` |
| `+` | 加 |
| `-` | 减 |
| `*` | 乘 |
| `/` | 除 |
| `%` | 取模 |
| `^` | 幂 |
| `>` | 大于 |
| `<` | 小于 |
| `=` | 等于 |
| `.` | 打印栈顶值并消耗 `( x -- )` |
| `_` | 行注释 |
| `'` | 字符串定界符 |

没有 `+=` `-=` `*=` `/=` `++` `--` 等复合运算符。
没有 `>=` `<=` `!=` `==` 等复合比较符。

### 待选符号（尚未分配用途）

```
( ) [ ] { } , | " `
```

## 三、系统原语

以 `:` 开头，后跟小写字母，作为一个完整词元。用于区分内部原语和用户定义字。

### 3.1 栈操作

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:dup` | 复制栈顶 | `( x -- x x )` |
| `:drop` | 丢弃栈顶 | `( x -- )` |
| `:swap` | 交换栈顶两个 | `( x y -- y x )` |
| `:over` | 复制第二个到栈顶 | `( x y -- x y x )` |
| `:rot` | 旋转前三个 | `( x y z -- y z x )` |

### 3.2 内存

| 原语 | 功能 | 栈效果 |
|------|------|--------|
| `:here` | 返回当前空闲地址 | `( -- addr )` |
| `:allot` | 分配 n 个单元 | `( n -- )` |
| `:type` | 打印字符串 | `( addr len -- )` |
| `:emit` | 输出单个字符 | `( c -- )` |
| `:stack` | 打印栈内所有内容（不消耗） | `( -- )` |

## 四、字定义

```
@ 字名 字定义体 ;
```

- `@` 开始定义，后跟字名
- `;` 结束定义，隐式返回
- `$` 可在定义体中显式提前返回

## 五、控制流

### 5.1 条件执行

`?` 和 `!` 只读取接下来的**一个词元**（用户字、系统原语、单符号、数字字面量、字符串字面量均可）。需要执行多条语句时，必须将代码块定义成字来调用。

单分支：

```
条件 ? 字
```

- 消耗栈顶标志，非 0 则执行该字，否则跳过

双分支：

```
条件 ? 真字 ! 假字
```

- 非 0 执行真字，0 执行假字

### 5.2 循环

`~` 无条件跳回当前字头部继续执行。

需要显式使用条件判断来控制循环退出，这是有意设计。

模式一：条件继续

```
@ example
    _ 循环体
    继续条件 ? ~
;
```

模式二：条件退出

```
@ example
    退出条件 ? $
    _ 循环体
    ~
;
```

模式三：条件退出带 else

```
@ example
    _ 循环体
    退出条件 ? 123 ! ~
;
```

## 六、内存

### 6.1 基本单位

以 cell（单元）为基本单位，不暴露字节概念。一个 cell 存储一个值（数字、地址等），底层宽度由实现决定，用户不感知。

### 6.2 读写

- `&` 读一个单元 `( addr -- value )`
- `#` 写一个单元 `( value addr -- )`

### 6.3 分配

采用线性分配模型，只增不减。

- `:here` 返回当前空闲地址 `( -- addr )`
- `:allot` 分配 n 个单元，移动 `:here` 指针 `( n -- )`

后续如需动态分配/释放（如 `:alloc` / `:free`），可独立添加，操作不同内存区域，与线性分配互不冲突。

## 七、程序入口

- 文件执行：`zona file.zona`，从头到尾顺序执行，顶层代码直接运行，定义字和执行代码可以混在一起
- REPL：`zona`（无参数），交互式逐行执行

两者本质相同：读取词元、解释执行。REPL 只是输入源从文件换成 stdin。

## 八、待定事项

- 数字类型系统（后缀精度等）
- 字符串模板
- 模块/命名空间
- 错误处理
- 栈类型系统：当前为无类型栈（传统 Forth 风格），数字和字符串混在栈上无法区分。后续可考虑带类型栈，使类型不匹配的操作（如数字与字符串相加）能报错
