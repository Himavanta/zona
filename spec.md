# Zona 语言规范

## 一、词法规则

- 空白分隔词元
- 全部使用单符号，没有组合复合符号语法

### 1.1 字的命名

- 只能使用字母和数字，第一位必须是字母
- 用户字、单符号、系统原语三者在词法层面天然不会冲突，因此没有关键字也没有保留字

### 1.2 字面量

字面量在运行时将常量压栈。

#### 数字

- 无特殊符号，自动处理
- 整数、浮点数、负数：`123` `1.3` `-5`
- 数字类型暂不细致讨论，后续可能通过后缀区分精度和类型，如 `3u8`

#### 布尔值

- `0` 为假，`1` 为真

#### 字符串

- 单引号包裹：`'hello'`
- 转义字符 `\`：`'\n'` `'hello \'tom\''`
- 字符串模板 `$`：暂不实现
- 在运行时压入字符串地址和长度

### 1.3 注释

- 行注释：`_`（从 `_` 到行尾）
- 没有块注释

## 二、单符号语法标记

固定，不可重定义。

| 符号 | 功能 |
|------|------|
| `@` | 开始定义字 |
| `;` | 结束定义字（隐式返回） |
| `$` | 显式中途返回 |
| `?` | 条件执行 |
| `!` | else 分支 |
| `~` | 跳回当前字头部 |
| `&` | 内存读 `( addr -- value )` |
| `#` | 内存写 `( value addr -- )` |
| `+` | 加 |
| `-` | 减 |
| `*` | 乘 |
| `/` | 除 |
| `%` | 取模 |
| `^` | 幂 |
| `>` | 大于 |
| `<` | 小于 |
| `=` | 等于 |
| `_` | 行注释 |
| `'` | 字符串定界符 |

没有 `+=` `-=` `*=` `/=` `++` `--` 等复合运算符。
没有 `>=` `<=` `!=` `==` 等复合比较符。

### 待选符号（尚未分配用途）

```
( ) [ ] { } , . | " `
```

## 三、系统原语

以 `:` 开头，后跟小写字母，作为一个完整词元。用于区分内部原语和用户定义字。

具体原语列表待定。已确认的方向：

- 栈操作：`:dup` `:drop` `:swap` `:over` `:rot` 等
- 内存管理：待定（可能需要分配和释放原语）
- IO：待定

## 四、字定义

```
@ 字名 字定义体 ;
```

- `@` 开始定义，后跟字名
- `;` 结束定义，隐式返回
- `$` 可在定义体中显式提前返回

## 五、控制流

### 5.1 条件执行

`?` 和 `!` 只读取接下来的**一个**字或符号。需要执行多条语句时，必须将代码块定义成字来调用。

单分支：

```
条件 ? 字
```

- 消耗栈顶标志，非 0 则执行该字，否则跳过

双分支：

```
条件 ? 真字 ! 假字
```

- 非 0 执行真字，0 执行假字

### 5.2 循环

`~` 无条件跳回当前字头部继续执行。

需要显式使用条件判断来控制循环退出，这是有意设计。

模式一：条件继续

```
@ example
    _ 循环体
    继续条件 ? ~
;
```

模式二：条件退出

```
@ example
    退出条件 ? $
    _ 循环体
    ~
;
```

模式三：条件退出带 else

```
@ example
    _ 循环体
    退出条件 ? 123 ! ~
;
```

## 六、内存

- `&` 读取指定地址的值 `( addr -- value )`
- `#` 向指定地址写入值 `( value addr -- )`
- 内存分配和释放的原语待定
