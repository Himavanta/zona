# Zona Next 语言规范

## 1. 设计哲学

- 极简语法 — 单字符符号，无复合符号
- 栈效应强制声明 — 编译期验证，运行时零开销
- 类型分离 — 整数与浮点在类型层面不同
- 面向硬件 — 栈是编译期概念，值应留在寄存器中

## 2. 词法

### 2.1 符号

V2 使用的符号：

```
@ ; : ? ! $ ~ + - * / % ^ > < = . _ '
```

| 符号 | 语义 | 备注 |
|------|------|------|
| `@` | 定义字 | |
| `;` | 结束定义 | |
| `:` | 签名分隔符 / 原语前缀 | 位置不冲突，见 3.1 |
| `?` | 条件执行 | |
| `!` | else 分支标记 | |
| `$` | 返回 | |
| `~` | 循环 | |
| `+` | 加（类型化） | |
| `-` | 减（类型化） | |
| `*` | 乘（类型化） | |
| `/` | 除（类型化） | |
| `%` | 取模（类型化） | |
| `^` | 幂（类型化） | |
| `>` | 大于（类型化） | |
| `<` | 小于（类型化） | |
| `=` | 等于（类型化） | |
| `.` | 成员访问 | 模块成员、结构体字段 |
| `_` | 注释（至行尾） | |
| `'` | 字符串定界符 | |

保留未分配的符号：`( ) [ ] { } , | " `` & #`

- `&` `#` — 预留给局部变量等未来特性
- `( ) [ ] { }` — 预留给错误处理、结构体、闭包等

### 2.2 字面量

```
35          整数字面量（i 类型）
0           整数 0（i 类型）
35.0        浮点字面量（d 类型）
0.0         浮点 0.0（d 类型）
'hello'     字符串字面量（压入 p i 两个值：地址和长度）
```

整数和浮点通过是否包含小数点区分。`35` 是整数，`35.0` 是浮点。

### 2.3 词元类型

- **符号** — 单字符，上表所列
- **字面量** — 整数、浮点、字符串
- **原语** — `:xxx` 格式，冒号前缀
- **用户字** — 字母开头，字母数字组成
- **注释** — `_` 至行尾

## 3. 类型系统

### 3.1 类型标记

| 标记 | 类型 | 大小 | ARM64 寄存器族 |
|------|------|------|----------------|
| `i` | 整数 | 64 位有符号 | x0-x28 |
| `d` | 浮点 | 64 位双精度 | d0-d31 |
| `p` | 指针 | 64 位地址 | x0-x28 |

### 3.2 栈效应声明

每个字定义**必须**包含栈效应声明，格式为 `输入类型:输出类型`，紧跟字名之后。

```
@ 名称 输入类型:输出类型 body ;
```

规则：
- `:` 前的字母序列是输入类型
- `:` 后的字母序列是输出类型
- 空输入或空输出省略字母即可
- 每个字母对应栈上的一个值，从左到右对应栈从底到顶

示例：

```
@ fib d:d ... ;         _ 消费1个浮点，产生1个浮点
@ neg d:d ... ;         _ 消费1个浮点，产生1个浮点
@ add dd:d + ;          _ 消费2个浮点，产生1个浮点
@ emit i: :drop ;       _ 消费1个整数，产生0个值
@ key :i 0 ;            _ 消费0个值，产生1个整数
@ hello : 'Hello' :type ; _ 消费0个值，产生0个值
```

### 3.3 栈效应验证

编译期强制验证。编译器模拟执行字的 body，追踪栈的增减，验证：
- body 执行后的栈净变化必须与声明一致
- 类型必须匹配 — 整数操作只能作用于 i 类型的值，浮点操作只能作用于 d 类型的值

验证失败则编译错误。

### 3.4 类型化操作

算术和比较符号根据操作数类型选择语义：

```
3 5 +           _ 整数加法，结果为 i 类型
3.0 5.0 +       _ 浮点加法，结果为 d 类型
3 5.0 +         _ 编译错误：类型不匹配
```

比较操作始终返回 i 类型（0 或 1）：

```
3 5 <           _ 整数比较，结果为 i
3.0 5.0 <       _ 浮点比较，结果为 i
```

### 3.5 `:` 的双重语义

`:` 在两种位置出现，编译器通过上下文区分：

- **签名分隔符** — 紧跟字名之后，body 之前。前后是类型字母或为空。
- **原语前缀** — body 内部，冒号后跟原语名。

```
@ fib d:d :dup 2.0 < ? $ :dup 1.0 - fib :swap 2.0 - fib + ;
      ^^^   ^^^^                    ^^^^       ^^^^^
     签名    原语                    原语       原语
```

人眼区分：`d:d` 紧凑字母串中间有冒号 → 签名；`:dup` 冒号开头跟词 → 原语。

## 4. 字定义

### 4.1 语法

```
@ 名称 输入类型:输出类型 body ;
```

组成部分：
1. `@` — 定义开始
2. 名称 — 用户字名（字母开头，字母数字组成）
3. 栈效应声明 — `输入类型:输出类型`
4. body — 零或多个词元
5. `;` — 定义结束

### 4.2 字的调用

body 中出现用户字名时，调用该字：

```
@ double d:d :dup + ;
@ quadruple d:d double double ;
```

### 4.3 字的重定义

后定义的同名字覆盖先前的。编译器在编译时使用最新定义。

## 5. 原语

原语以 `:` 前缀标识，与用户字词法层面不冲突。

### 5.1 栈操作

| 原语 | 签名 | 语义 |
|------|------|------|
| `:dup` | 单类型 `x:xx` | 复制栈顶 |
| `:drop` | 单类型 `x:` | 丢弃栈顶 |
| `:swap` | 双类型 `xy:yx` | 交换栈顶两个 |
| `:over` | 双类型 `xy:xyx` | 复制次栈顶到栈顶 |
| `:rot` | 三类型 `xyz:yzx` | 旋转栈顶三个 |

注：栈操作原语的签名随操作数类型变化。编译器根据上下文推断具体类型。

### 5.2 I/O

| 原语 | 签名 | 语义 |
|------|------|------|
| `:print` | `i:` | 打印整数 |
| `:print` | `d:` | 打印浮点 |
| `:emit` | `i:` | 输出字符 |
| `:key` | `:i` | 读入字符 |

### 5.3 内存

| 原语 | 签名 | 语义 |
|------|------|------|
| `:alloc` | `i:p` | 分配内存，返回真实指针 |
| `:free` | `p:` | 释放内存 |
| `:peek` | `p:i` | 从指针读整数 |
| `:poke` | `ip:` | 往指针写整数 |

### 5.4 字符串

| 原语 | 签名 | 语义 |
|------|------|------|
| `:type` | `pi:` | 打印字符串（地址+长度） |

### 5.5 系统

| 原语 | 签名 | 语义 |
|------|------|------|
| `:time` | `:i` | 当前时间戳 |
| `:rand` | `i:i` | 随机数 |
| `:exit` | `i:` | 退出程序 |

## 6. 控制流

### 6.1 条件执行 `?`

`?` 读取栈顶值（i 类型），非零为真。

**单分支** — `? 词元`

```
:dup 2.0 < ? $          _ 如果小于2.0则返回
```

**双分支** — `? 词元 ! 词元`

```
:dup 0.0 < ? neg !     _ 如果小于0则取反，否则不变
```

`!` 标记 else 分支。

### 6.2 返回 `$`

立即退出当前字，返回调用者。

### 6.3 循环 `~`

跳回当前字的开头，重新执行。

```
@ countdown i: :dup 0 = ? $ :print 1 - ~ ;
```

常用模式：

```
? $ ~      _ 条件退出循环（如果为真则返回，否则继续循环）
? ~        _ 条件继续循环（如果为真则跳回开头）
```

## 7. 内联

- 字 body ≤ 8 个词元且非递归 → 默认内联
- 递归字或长字 → 不内联
- 内联在解释器中是字节码展开，在编译器中是机器码展开

## 8. 成员访问 `.`

`.` 用于访问模块成员和结构体字段：

```
io.open         _ 模块成员
task.pid        _ 结构体字段
```

打印使用 `:print` 原语，不再使用 `.`。

## 9. 注释

`_` 至行尾为注释：

```
_ 这是一行注释
@ fib d:d _ 计算斐波那契数 :dup 2.0 < ? $ :dup 1.0 - fib :swap 2.0 - fib + ;
```

## 10. 完整示例

```
_ 斐波那契数列
@ fib d:d :dup 2.0 < ? $ :dup 1.0 - fib :swap 2.0 - fib + ;
35.0 fib :print

_ 阶乘
@ fact d:d :dup 1.0 < ? $ :dup 1.0 - fact * ;
10.0 fact :print

_ 倒计时
@ countdown i: :dup 0 = ? $ :print 1 - ~ ;
5 countdown

_ 条件分支
@ abs d:d :dup 0.0 < ? 0.0 :swap - ! ;
-3.0 abs :print

_ 使用原语
@ greet : 'Hello, World!' :type 10 :emit ;
greet
```

## 11. 与 V1 的差异总结

| 特性 | V1 | V2 |
|------|----|----|
| 栈效应声明 | 无 | 强制（`输入:输出`） |
| 数字类型 | 统一 double | 整数 `i` / 浮点 `d` 分离 |
| 类型化操作 | 无（全按 double） | 算术/比较按操作数类型分派 |
| 原语前缀 | `:` | `:`（不变） |
| 签名分隔 | 无 | `:` |
| `!` | else 分支 | else 分支（不变） |
| `.` | 打印 / 成员访问 | 仅成员访问 |
| 打印 | `.` | `:print` 原语 |
| `&` `#` | 内存读 / 内存写 | 释放（改为 `:peek` `:poke` 原语） |
| 指针 | double 编码的偏移量 | 真实地址（p 类型） |
| 内联 | 无 | 默认内联短词 |
| 函数参数 | 全局栈传递 | 寄存器传递（编译器） |
