# Zona Next 语言规范

## 1. 设计哲学

- 极简语法 — 单字符符号，无复合符号
- 栈效应强制声明 — 编译期验证，运行时零开销
- 类型分离 — 整数与浮点在类型层面不同
- 面向硬件 — 栈是编译期概念，值应留在寄存器中

## 2. 词法

### 2.1 符号

V2 使用的符号：

```
@ ; : ? ! $ ~ + - * / % ^ > < = . _ '
```

| 符号 | 语义 | 备注 |
|------|------|------|
| `@` | 定义字 | |
| `;` | 结束定义 | |
| `:` | 签名分隔符 / 原语前缀 | 位置不冲突，见 3.1 |
| `?` | 条件执行 | |
| `!` | else 分支标记 | |
| `$` | 返回 | |
| `~` | 循环 | |
| `+` | 加（类型化） | |
| `-` | 减（类型化） | |
| `*` | 乘（类型化） | |
| `/` | 除（类型化） | |
| `%` | 取模（类型化） | |
| `^` | 幂（类型化） | |
| `>` | 大于（类型化） | |
| `<` | 小于（类型化） | |
| `=` | 等于（类型化） | |
| `.` | 成员访问 | 模块成员、结构体字段 |
| `_` | 注释（至行尾） | |
| `'` | 字符串定界符 | |

保留未分配的符号：`( ) [ ] { } , | " `` & #`

- `&` `#` — 预留给局部变量等未来特性
- `( ) [ ] { }` — 预留给错误处理、结构体、闭包等

### 2.2 字面量

```
35          整数字面量（l 类型）
0           整数 0（l 类型）
35.0        浮点字面量（d 类型）
0.0         浮点 0.0（d 类型）
'hello'     字符串字面量（压入 p l 两个值：地址和长度）
```

整数和浮点通过是否包含小数点区分。`35` 是整数，`35.0` 是浮点。

### 2.3 词元类型

- **符号** — 单字符，上表所列
- **字面量** — 整数、浮点、字符串
- **原语** — `:xxx` 格式，冒号前缀
- **用户字** — 字母开头，字母数字组成
- **注释** — `_` 至行尾

## 3. 类型系统

### 3.1 统一类型词汇表

Zona 使用一套类型词汇表，不同上下文使用不同子集：

```
完整词汇表：b  h  w  l  s  d  p  p*
大小：      1  2  4  8  4  8  8  8 字节
含义：     字节 半字 字 长 单精 双精 指针 串指针
```

| 类型 | 大小 | 含义 | 栈效应 | `:struct` | `:bind` |
|------|------|------|:------:|:---------:|:-------:|
| `b` | 1 字节 | 字节 | | ✅ | |
| `h` | 2 字节 | 半字 | | ✅ | |
| `w` | 4 字节 | 32 位整数 | | ✅ | ✅ |
| `l` | 8 字节 | 64 位整数 | ✅ | ✅ | ✅ |
| `s` | 4 字节 | 单精度浮点 | | ✅ | ✅ |
| `d` | 8 字节 | 双精度浮点 | ✅ | ✅ | ✅ |
| `p` | 8 字节 | 指针 | ✅ | ✅ | ✅ |
| `p*` | 8 字节 | 字符串指针 | | | ✅ |

子集划分的原因：
- **栈效应**只用 `l` `d` `p` — 栈是 64 位槽，`w`/`s` 与 `l`/`d` 在栈上等价
- **`:struct`** 用 `b`-`p`（不含 `p*`）— 精确内存布局
- **`:bind`** 用 `w` `l` `s` `d` `p` `p*` — FFI 边界需要精确类型 + 字符串指针特殊处理

`p*` 的特殊性：字符串在 Zona 栈上是两个值（地址 + 长度），`p*` 在栈效应中展开为 `pl`。`p*` 仅用于 `:bind` 中标注 `char*` 参数/返回值，告诉编译器在 FFI 边界做自动字符串转换。

与 QBE IL 的对应：`b`↔`b`，`h`↔`h`，`w`↔`w`，`l`↔`l`，`s`↔`s`，`d`↔`d`，`p`↔`l`，`p*`↔`l`+字符串转换。直接映射，无中间层。

### 3.2 栈效应声明

每个字定义**必须**包含栈效应声明，格式为 `输入类型:输出类型`，紧跟字名之后。

```
@ 名称 输入类型:输出类型 body ;
```

规则：
- `:` 前的字母序列是输入类型
- `:` 后的字母序列是输出类型
- 空输入或空输出省略字母即可
- 每个字母对应栈上的一个值，从左到右对应栈从底到顶
- 只使用 `l` `d` `p` 三个类型标记

示例：

```
@ fib d:d ... ;          _ 消费1个双精度，产生1个双精度
@ neg d:d ... ;          _ 消费1个双精度，产生1个双精度
@ add dd:d + ;           _ 消费2个双精度，产生1个双精度
@ emit l: :drop ;        _ 消费1个长整数，产生0个值
@ key :l 0 ;             _ 消费0个值，产生1个长整数
@ hello : 'Hello' :type ; _ 消费0个值，产生0个值
```

### 3.3 栈效应验证

编译期强制验证。编译器模拟执行字的 body，追踪栈的增减，验证：
- body 执行后的栈净变化必须与声明一致
- 类型必须匹配 — 整数操作只能作用于 l 类型的值，浮点操作只能作用于 d 类型的值

验证失败则编译错误。

### 3.4 类型化操作

算术和比较符号根据操作数类型选择语义：

```
3 5 +           _ 整数加法，结果为 l 类型
3.0 5.0 +       _ 浮点加法，结果为 d 类型
3 5.0 +         _ 编译错误：类型不匹配
```

比较操作始终返回 l 类型（0 或 1）：

```
3 5 <           _ 整数比较，结果为 l
3.0 5.0 <       _ 浮点比较，结果为 l
```

### 3.5 `:` 的双重语义

`:` 在两种位置出现，编译器通过上下文区分：

- **签名分隔符** — 紧跟字名之后，body 之前。前后是类型字母或为空。
- **原语前缀** — body 内部，冒号后跟原语名。

```
@ fib d:d :dup 2.0 < ? $ :dup 1.0 - fib :swap 2.0 - fib + ;
      ^^^   ^^^^                    ^^^^       ^^^^^
     签名    原语                    原语       原语
```

人眼区分：`d:d` 紧凑字母串中间有冒号 → 签名；`:dup` 冒号开头跟词 → 原语。

### 3.6 栈物理模型

运行时使用统一栈，每个槽位 64 位，无运行时类型标签：

```c
uint64_t stack[2048];
int sp;
```

编译器/字节码生成器根据栈效应签名追踪每个位置的类型，生成类型化指令。运行时无需类型检查，类型信息已编码在指令中。

- `3 5 +` → `PUSH_L 3, PUSH_L 5, ADD_L` — 整数路径
- `3.0 5.0 +` → `PUSH_D 3.0, PUSH_D 5.0, ADD_D` — 浮点路径
- `:swap` 编译时根据栈效应签名生成正确字节码 — `dd` 生成 `SWAP_D`，`ll` 生成 `SWAP_L`，`ld` 生成 `SWAP_LD`（跨类型交换）
- `:dup` `:drop` 等原语不需要区分类型 — 64 位值的复制/丢弃是通用操作

选择此模型的原因：
- 运行时零开销 — 无标签检查，无内存浪费
- 和编译器目标一致 — 编译期追踪类型，ARM64 代码生成时整数走 x 寄存器，浮点走 d 寄存器
- 语义简洁 — `:swap` 仍然是"交换栈顶两个值"，和 Forth 传统一致

### 3.7 控制流汇合的栈类型约束

`?` 的两个分支汇合时，栈的类型布局必须一致。如果不一致，编译错误。

**单分支** `? word`：执行 `word` 后（真）和跳过 `word` 后（假）的栈类型布局必须一致。

**双分支** `? word1 ! word2`：执行 `word1` 后（真）和执行 `word2` 后（假）的栈类型布局必须一致。

**返回豁免**：如果分支中使用了 `$`（返回），该分支不参与汇合约束 — 返回直接退出当前字，没有汇合点。

合法示例：

```
@ abs d:d
  :dup 0.0 < ? 0.0 :swap - ! ;    _ 真/假分支都产出 d ✓
```

非法示例：

```
@ bad d:d
  :dup 0.0 < ? :drop 1 ! ;         _ 真分支产出 l，假分支产出 d → 编译错误
```

这条约束是特性而非限制：确保编译器在每个汇合点能确定栈上每个位置的类型，防止类型混乱的 bug。

## 4. 字定义

### 4.1 语法

```
@ 名称 输入类型:输出类型 body ;
```

组成部分：
1. `@` — 定义开始
2. 名称 — 用户字名（字母开头，字母数字组成）
3. 栈效应声明 — `输入类型:输出类型`
4. body — 零或多个词元
5. `;` — 定义结束

### 4.2 字的调用

body 中出现用户字名时，调用该字：

```
@ double d:d :dup + ;
@ quadruple d:d double double ;
```

### 4.3 字的重定义

后定义的同名字覆盖先前的。编译器在编译时使用最新定义。

## 5. 原语

原语以 `:` 前缀标识，与用户字词法层面不冲突。

### 5.1 栈操作

| 原语 | 签名 | 语义 |
|------|------|------|
| `:dup` | 单类型 `x:xx` | 复制栈顶 |
| `:drop` | 单类型 `x:` | 丢弃栈顶 |
| `:swap` | 双类型 `xy:yx` | 交换栈顶两个 |
| `:over` | 双类型 `xy:xyx` | 复制次栈顶到栈顶 |
| `:rot` | 三类型 `xyz:yzx` | 旋转栈顶三个 |

注：栈操作原语的签名随操作数类型变化。编译器根据上下文推断具体类型。

### 5.2 I/O

| 原语 | 签名 | 语义 |
|------|------|------|
| `:print` | `l:` | 打印整数 |
| `:print` | `d:` | 打印浮点 |
| `:emit` | `l:` | 输出字符 |
| `:key` | `:l` | 读入字符 |

### 5.3 内存

| 原语 | 签名 | 语义 |
|------|------|------|
| `:alloc` | `l:p` | 分配内存，返回真实指针 |
| `:free` | `p:` | 释放内存 |
| `:peek` | `p:l` | 从指针读整数 |
| `:poke` | `lp:` | 往指针写整数 |

### 5.4 字符串

| 原语 | 签名 | 语义 |
|------|------|------|
| `:type` | `pl:` | 打印字符串（地址+长度） |

### 5.5 系统

| 原语 | 签名 | 语义 |
|------|------|------|
| `:time` | `:l` | 当前时间戳 |
| `:rand` | `l:l` | 随机数 |
| `:exit` | `l:` | 退出程序 |

## 6. 控制流

### 6.1 条件执行 `?`

`?` 读取栈顶值（l 类型），非零为真。**每个分支只执行一个词元**。多词元操作需定义辅助字。

**单分支** — `? 词元`

```
:dup 2.0 < ? $          _ 如果小于2.0则返回
```

**双分支** — `? 词元 ! 词元`

```
:dup 0.0 < ? neg !     _ 如果小于0则取反，否则不变
```

`!` 标记 else 分支。

**栈类型汇合约束** — `?` 分支汇合时栈的类型布局必须一致，详见 3.7。

### 6.2 返回 `$`

立即退出当前字，返回调用者。

### 6.3 循环 `~`

跳回当前字的开头，重新执行。

```
@ countdown l: :dup 0 = ? $ :dup :print 1 - ~ ;
```

常用模式：

```
? $ ~      _ 条件退出循环（如果为真则返回，否则继续循环）
? ~        _ 条件继续循环（如果为真则跳回开头）
```

**循环的栈类型约束** — `~` 跳回当前字开头重新执行，因此 `~` 处的栈类型布局必须与字入口处（消费输入参数后）的栈类型布局一致。否则编译错误。

合法示例：

```
@ countdown l: :dup 0 = ? $ :dup :print 1 - ~ ;
_                        入口: l  ~处: l ✓
```

非法示例：

```
@ bad l: 1 + ~ ;
_          入口: 空(已消费l)  ~处: l → 编译错误
```

## 7. 内联

- 字 body ≤ 8 个词元且非递归 → 默认内联
- 递归字或长字 → 不内联
- 内联在解释器中是字节码展开，在编译器中是机器码展开

## 8. 成员访问 `.`

`.` 用于访问模块成员和结构体字段：

```
io.open         _ 模块成员
task.pid        _ 结构体字段
```

打印使用 `:print` 原语，不再使用 `.`。

## 9. 结构体 🔮 远期暂定

> 结构体语法暂定，实现推迟到编译器 + FFI 阶段之后。结构体与 FFI 紧密相关（C 结构体内存布局、按值/按指针传递），在 FFI 能力就绪后再实现。

### 9.1 声明

`:struct` 在顶层声明结构体类型，独占一行，无结束符号：

```
:struct Point x d y d
:struct Color r b g b b b a b
:struct Rect x s y s w s h s
```

格式：`:struct` + 名称 + 字段名/类型对

规则：
- 必须在顶层声明，不能在字定义（`@ ... ;`）内声明
- 字段必须有名字和类型标记（使用统一类型词汇表，不含 `p*`）
- 与 `:use` `:bind` 同级，属于编译期声明

字段类型使用统一类型词汇表中的 `b` `h` `w` `l` `s` `d` `p`，直接对应 QBE aggregate 定义。偏移量由编译器根据类型大小自动计算：

| 字段类型 | 大小 | 对齐 |
|----------|------|------|
| `b` | 1 字节 | 1 |
| `h` | 2 字节 | 2 |
| `w` | 4 字节 | 4 |
| `l` | 8 字节 | 8 |
| `s` | 4 字节 | 4 |
| `d` | 8 字节 | 8 |
| `p` | 8 字节 | 8 |

字段读取时按类型映射到栈效应标记：`b`/`h`/`w`/`l`/`p` → `l`，`s`/`d` → `d`。

### 9.2 自动生成的字

声明 `:struct Point x d y d` 后，编译器自动生成：

| 字 | 签名 | 语义 |
|------|------|------|
| `Point.new` | `:p` | 在堆上分配内存并零初始化，返回指针 |
| `Point.free` | `p:` | 释放内存 |

### 9.3 字段访问

字段访问使用 `.` 成员访问语法，与模块成员访问统一为单一 token（`T_MEMBER`）：

```
Point.new :dup Point.x      _ 构造 + 读取 x 字段（栈: p → p d）
```

`Point.x` 解析为 `T_MEMBER` 类型 token。编译器根据 `Point` 的 `:struct` 声明确定其为结构体字段访问，生成相应的字段读取代码。

自动生成字段读取字：

| 字 | 签名 | 语义 |
|------|------|------|
| `Point.x` | `p:d` | 读取字段 x（偏移量由 `:struct` 声明计算） |
| `Point.y` | `p:d` | 读取字段 y |

字段写入同样自动生成，命名在字段名后加 `!`（如 `Point.x!`）。具体是否与 else 分支的独立 `!` 符号冲突，取决于 tokenizer 将 `xxx.yyy!` 解析为单一 `T_MEMBER` 还是断开为 `xxx.yyy` + `!`。实现时确定。

## 10. 注释

`_` 至行尾为注释：

```
_ 这是一行注释
@ fib d:d _ 计算斐波那契数 :dup 2.0 < ? $ :dup 1.0 - fib :swap 2.0 - fib + ;
```

## 11. 完整示例

```
_ 斐波那契数列
@ fib d:d :dup 2.0 < ? $ :dup 1.0 - fib :swap 2.0 - fib + ;
35.0 fib :print

_ 阶乘
@ fact d:d :dup 1.0 < ? $ :dup 1.0 - fact * ;
10.0 fact :print

_ 倒计时
@ countdown l: :dup 0 = ? $ :dup :print 1 - ~ ;
5 countdown

_ 条件分支
@ neg d:d 0.0 :swap - ;
@ abs d:d :dup 0.0 < ? neg ! ;
-3.0 abs :print

_ 使用原语
@ greet : 'Hello, World!' :type 10 :emit ;
greet
```

## 12. 与 V1 的差异总结

| 特性 | V1 | V2 |
|------|----|----|
| 栈效应声明 | 无 | 强制（`输入:输出`） |
| 数字类型 | 统一 double | 整数 `l` / 浮点 `d` 分离 |
| 类型化操作 | 无（全按 double） | 算术/比较按操作数类型分派 |
| 原语前缀 | `:` | `:`（不变） |
| 签名分隔 | 无 | `:` |
| `!` | else 分支 | else 分支（不变） |
| `.` | 打印 / 成员访问 | 仅成员访问 |
| 打印 | `.` | `:print` 原语 |
| `&` `#` | 内存读 / 内存写 | 释放（改为 `:peek` `:poke` 原语） |
| 指针 | double 编码的偏移量 | 真实地址（p 类型） |
| 内联 | 无 | 默认内联短词 |
| 函数参数 | 全局栈传递 | 寄存器传递（编译器） |
| 栈物理模型 | 统一 double 栈 | 统一 64 位栈 + 编译期类型追踪 |
| 结构体 | 无 | `:struct` 顶层声明，自动生成构造/析构/字段访问 |
