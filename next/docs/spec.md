# Zona 语言规范

> 最终版。基于实际实现的编译器 (`zonac.c`)。
> 设计哲学：性能优先，栈效应承担类型区分的职责。

## 一、词法

- 空白分隔词元
- `_` 至行尾为注释

### 1.1 字的命名

- 只能使用字母和数字，第一位必须是字母
- 无关键字，无保留字——用户字与单符号和系统原语词法层面不冲突

### 1.2 符号

```
@ ; : ? ! $ ~ + - * / % ^ > < = . _ ' & #
```

| 符号 | 语义 |
|------|------|
| `@` | 开始字定义 |
| `;` | 结束字定义 |
| `:` | 栈效应分隔符 / 原语前缀（见 §1.5） |
| `?` | 条件执行（消费栈顶 l，非零为真） |
| `!` | else 分支标记 |
| `$` | 显式返回 |
| `~` | 跳回当前字头部（循环） |
| `+ - * / % ^` | 算术（类型化） |
| `> < =` | 比较（类型化，返回 l） |
| `&` | 内存读 `( p -- l|d )` |
| `#` | 内存写 `( l|d p -- )` |
| `.` | 打印（有空格）/ 成员访问（无空格） |
| `_` | 行注释 |
| `'` | 字符串定界符 |

**保留未分配**：`( ) [ ] { } , | " \``

### 1.3 字面量

数字类型由**小数点**决定——不含小数点即为整数，含小数点即为浮点。无需后缀，语法层面一目了然。

**整数**（l 类型）：不含小数点。`35` `0` `-5`

**浮点**（d 类型）：含小数点。`35.0` `0.0` `-5.0`。QBE 要求浮点常量格式 `d_X.0`，编译器内部用 `%g` 输出保证正确精度。

**字符串**：单引号包裹。`'hello'`。压入 `p l` 两个值——data section 的 C 字符串指针 (null 结尾，保证 FFI 兼容) + 编译时长度。转义支持 `\n` `\t` `\\` `\'`。

### 1.4 词元类型

| 类型 | 例子 | TokenType |
|------|------|-----------|
| 数字 | `35` `35.0` | T_NUM |
| 字符串 | `'hello'` | T_STR |
| 符号 | `+` `?` `@` `:` | T_SYM |
| 系统原语 | `:dup` `:type` | T_PRIM |
| 用户字 | `fib` `abs` | T_WORD |
| 成员访问 | `math.abs` | T_MEMBER |

### 1.5 栈效应 `:` 词法歧义

`:` 根据上下文区分：`d:d` 中间有冒号 → 签名分隔符；`:dup` 冒号开头跟词 → 原语前缀。tokenizer 将独立 `:` 作为 T_SYM，`:xxx` 作为 T_PRIM。

## 二、类型系统

栈效应子集（8 字节栈槽）：`l` `d` `p`

| 类型 | 含义 | 字面量 |
|------|------|--------|
| `l` | 64位整数 | `35` |
| `d` | 64位浮点（IEEE 754） | `35.0` |
| `p` | 指针 / 字符串地址 | `'hello'` `:alloc` |

**类型化操作**：算术和比较根据操作数类型选择语义。`3 5 +` 整数加法，`3.0 5.0 +` 浮点加法，类型不匹配编译错误。比较始终返回 `l`（0 或 1）。

## 三、栈效应声明

每个字定义必须包含栈效应，格式为 `输入类型:输出类型`，紧跟字名之后。

```
@ 名称 输入类型:输出类型 body ;
```

- `:` 前为输入类型序列，后为输出类型序列
- 每个字母对应栈上一个值，从左到右对应栈从底到顶
- 只使用 `l d p` 三个类型标记
- 空输入/空输出省略字母

```
@ fib d:d ... ;       _ 1 个 d 入，1 个 d 出
@ add ll:l + ;        _ 2 个 l 入，1 个 l 出
@ greet : 'hi' :type ; _ 0 入，0 出
```

**重定义**：同名字覆盖先定义，编译器用最新定义。

## 四、控制流

### 4.1 条件 `?`

消费栈顶 l 类型值。每个分支只执行**一个词元**。

```
条件 ? 真词元                _ 单分支
条件 ? 真词元 ! 假词元        _ 双分支
```

`!` 标记 else 分支。分支汇合时栈类型布局必须一致，使用 `$` 返回的分支豁免汇合约束。

### 4.2 返回 `$`

立即退出当前字。函数返回结果在虚栈上，由编译器自动生成 `ret` 指令。

### 4.3 循环 `~`

跳回当前字头部。`~` 处栈类型必须与字入口处（消费输入参数后）一致，否则编译错误。

## 五、字符串表示

`'hello'` 压入 `p l` 两个值：

- `p`：data section 中 C 字符串的指针（null 结尾，保证 FFI 兼容性）。
- `l`：编译时已知的字符串长度。

`:type` 签名 `pl:`，用 `printf("%.*s", len, ptr)` 输出，不依赖 `\0`。FFI `:bind` 传 `char*` 参数时，字符串指针直接传寄存器，零开销。

**为何后置长度而非前置？** 指针即数据头，传给 C 函数时不需要偏移。长度在栈上独立存在，O(1) 访问。字符串可含 `\0`。

**为何 null 结尾？** 保证与所有 C 函数（`fopen`、`puts` 等）的兼容性。一个字节代价，FFI 边界零转换开销。

## 六、内存读写

`&` 读内存，`#` 写内存。固定 8 字节宽度。

| 符号 | 签名 | 语义 |
|------|------|------|
| `&` | `p:l` 或 `p:d` | 读 8 字节 |
| `#` | `lp:` 或 `dp:` | 写 8 字节 |

编译器生成 QBE `loadl`/`loadd`/`storel`/`stored`。

**为何不做引用封装？** 引用模型（`{addr, value}` 结构体）引入不透明的中间层，调试时无法直接看到引用的地址和值，且对逐字节操作不透明。高频的 `&` `#` 应保持直接和零开销。引用能力可在标准库层实现。

**窄宽度（1/2/4字节）读写**：留给未来的结构体字段访问。`:struct` 声明中字段类型可为 `b h w`，编译器自动生成正确宽度的 load/store。

## 七、原语

系统原语以 `:` 前缀标识，与用户字词法层面不冲突。

### 7.1 栈操作

| 原语 | 签名 | 语义 |
|------|------|------|
| `:dup` | `x:xx` | 复制栈顶 |
| `:drop` | `x:` | 丢弃栈顶 |
| `:swap` | `xy:yx` | 交换栈顶两个 |
| `:over` | `xy:xyx` | 复制次栈顶 |
| `:rot` | `xyz:yzx` | 旋转栈顶三个 |

### 7.2 I/O

| 原语 | 签名 | 语义 |
|------|------|------|
| `:print` | `l:` 或 `d:` | 打印栈顶并消耗 |
| `.` | `l:` 或 `d:` | 同上（有空格时） |
| `:type` | `pl:` | 输出字符串（`printf("%.*s")`） |
| `:emit` | `l:` | 输出字符 |
| `:key` | `:l` | 读入字符 |

### 7.3 内存

| 原语 | 签名 | 语义 |
|------|------|------|
| `:alloc` | `l:p` | `malloc(n)`，返回真实指针 |
| `:free` | `p:` | `free(ptr)` |

### 7.4 系统

| 原语 | 签名 | 语义 |
|------|------|------|
| `:time` | `:l` | `time(NULL)` |
| `:rand` | `l:l` | `rand() % n` |
| `:exit` | `l:` | `exit(code)` |
| `:argc` | `:l` | 命令行参数个数 |
| `:argv` | `l:pl` | 第 n 个参数，压入字符串指针和长度 |

### 7.5 编译时原语

| 原语 | 语义 |
|------|------|
| `:use` | 引入文件 |
| `:bind` | FFI 绑定（待实现） |

### 7.6 调试与内省

| 原语 | 签名 | 语义 |
|------|------|------|

| `:words` | `:` | 列出所有已定义字及其签名 |
| `:see` | `p:` | 查看指定字的定义体和签名 |

## 八、程序入口和执行模型

### 8.1 编译器模式

```
zonac <file.zona> [-o output]
```

`zonac` 是编译器入口。编译管线：

1. 读取源文件，tokenize
2. 第一遍：收集所有 `@ ... ;` 字定义、字符串字面量，处理 `:use` 加载模块
3. 第二遍：为每个字生成带类型签名的 QBE 函数，生成 `$main` 函数体（含顶层松散代码执行）
4. QBE 将 IR 编译为汇编
5. `cc` 汇编并链接为可执行文件（自动链接 `-lm`）

编译产物是原生可执行文件，无运行时依赖。

### 8.2 松散代码执行

文件顶层中不在 `@ ... ;` 定义体内的代码称为松散代码，编译到 `$main` 函数中，在程序启动时顺序执行。`$main` 的参数为 `( argc argv )`，返回值作为程序退出码。

### 8.3 参数传递约定

程序启动时通过全局变量暴露命令行参数：

- `$prog_argc`（w 类型）：程序参数个数。第一个参数为源文件名，后续为程序自定义参数。
- `$prog_argv`（l 类型）：指向 `argv` 指针数组的指针。

`:argc` 和 `:argv` 原语封装了对这两个全局变量的访问。

### 8.4 构建流程

```
.zona → tokenizer → 两遍编译 → QBE IR (.ssa) → qbe → 汇编 (.s) → cc → 可执行文件
```

生成的可执行文件链接 libc/libm，可直接运行。

## 九、模块系统

### 9.1 语法

```
:use name 'path'    _ 命名导入
```

通过 `name.word` 成员访问语法引用模块中的字。

### 9.2 成员访问

`name.word` 解析为 T_MEMBER token。嵌套模块链式访问：`b.c.word` 逐层解析 `.`，除最后一个段外均为模块名，在已加载模块中查找。

### 9.3 嵌套导入

模块文件内部可以使用 `:use` 引入其他模块。嵌套的模块可通过链式 `.` 访问——如果模块 `a` 引入了模块 `b`，而 `b` 引入了模块 `c`，则可以通过 `b.c.word` 访问 `c` 中的字。

### 9.4 去重

基于绝对路径。同一文件只加载一次（`already_used()` 检查），避免循环引用。

### 9.5 路径解析

`:use` 中的相对路径基于当前源文件所在目录解析。编译器通过 `current_dir` 全局变量追踪当前文件目录，`:use` 时路径基于此解析。

### 9.6 命名空间

全部导出，命名空间即封装。模块中所有字都可通过 `模块名.词名` 访问。别名通过普通字定义实现：

```
@ add3 ll:l math.add ;
```

### 9.7 行格式规则

`:use` 必须遵循以下约束：

- **行首起始**：必须是该行的第一个词元
- **独占一行**：一行内不可混入其他 token
- **字定义体外**：不能出现在 `@ ... ;` 定义体内部
- **行尾可注释**：`_` 注释可以跟在路径字符串后

```
:use math './math.zona'             _ ✅ 正确
:use math './math.zona' :use other './other.zona'  _ ❌ 一行多个
@ foo : 1 ; :use math './math.zona'                 _ ❌ 不在行首
@ foo :use math './math.zona' ;                     _ ❌ 在定义体内
```

编译器在遇到 `:use` 时按行提取 token，仅当该行首 token 为 `:use` 时才做导入处理。

## 十、结构体成员访问 ⏳

> 远期目标，当前仅预留设计方向，未实现。

### 10.1 基础语法

```
:struct Point l l             _ 声明结构体：两个 l 字段
p.x .                         _ 访问 p 的 x 字段
p.y .                         _ 访问 p 的 y 字段
```

### 10.2 嵌套结构体

`.` 链式访问天然支持嵌套：

```
:struct Pos d d
:struct Player Pos l
player.pos.x .                _ 深度读取
```

### 10.3 窄宽度字段

`:struct` 声明中字段类型除 `l d p` 外，还可使用窄宽度类型标记：

| 类型标记 | C 对应 | 宽度 |
|---------|--------|------|
| `b` | `char` | 1 字节 |
| `h` | `short` | 2 字节 |
| `w` | `int` | 4 字节 |
| `l` | `long` | 8 字节 |
| `d` | `double` | 8 字节 |
| `p` | `void*` | 8 字节 |

编译器根据字段类型自动生成正确宽度的 load/store，参考 `&` `#` 的窄宽度扩展（见 §六）。

### 10.4 C 互操作

与 `:bind` 配合使用，C 结构体可以直接映射：

```
:struct Color w w w w         _ raylib 的 Color: r g b a
:bind drawCircle 'DrawCircle' void int int float Color
```

### 10.5 非规范字段名

当 C 结构体字段名不符合 Zona 命名规则时（如含下划线），使用 `'...'` 包裹：

```
:struct Image w w p
img.'width' .                 _ C 结构体的 width 字段
img.'data_ptr' .              _ C 的 data_ptr 字段
```

`.` 紧贴字名，`'str'` 紧贴 `.`，无空格。

### 10.6 与模块访问的统一

模块和结构体本质都是"命名容器"，`.` 统一表达"从容器中取出命名成员"：

| 语境 | 容器 | 示例 |
|------|------|------|
| `math.abs` | 模块 | 从 math 模块取 abs |
| `point.x` | 结构体 | 从 point 结构体取 x 字段 |
| `net.http.get` | 嵌套模块 | 深度模块访问 |
| `player.pos.x` | 嵌套结构体 | 深度字段访问 |

## 十一、FFI 设计

### 11.1 语法

```
:bind zonaName 'cName' retType paramTypes...
```

声明外部 C 函数绑定。编译器记录 C 函数名和类型签名，在调用点按 AAPCS64 规则生成调用指令。

```
:bind initWindow 'InitWindow' void int int char*
:bind getWidth 'GetScreenWidth' int
:bind drawCircle 'DrawCircle' void int int float Color
:bind getenv 'getenv' char* char*
:bind cmalloc 'malloc' void* long
```

### 11.2 类型映射

使用 C 标准类型名称，指针类型用 `*` 后缀。编译器映射到内部 `l d p` 类型：

| zona 类型名 | C 类型 | 内部映射 | 说明 |
|------------|--------|---------|------|
| `void` | void | — | 仅用于返回类型 |
| `char` | char | l | 1 字节整数 |
| `short` | short | l | 2 字节整数 |
| `int` | int | l | 4 字节整数 |
| `long` | long | l | 8 字节整数 |
| `float` | float | d | 单精度浮点 |
| `double` | double | d | 双精度浮点 |
| `char*` | const char* | p | 字符串，自动传递指针 |
| `void*` | void* | p | 通用指针 |
| `int*` 等 | 对应指针 | p | 任意类型指针 |
| `Color` 等 | 结构体 | — | 按值传递（需先 `:struct` 声明，见 §十）|

`char*` 和 `void*` 中的 `*` 在 `:bind` 行内特殊处理——tokenizer 会将其拆为两个词元，`:bind` 解析时自动合并。

### 11.3 行格式规则

与 `:use` 一致（见 §9.7）：

- **行首起始**：必须是该行的第一个词元
- **独占一行**：一行内不可混入其他 token
- **字定义体外**：不能出现在 `@ ... ;` 定义体内部
- **行尾可注释**：`_` 注释可以跟在参数类型后

## 十二、设计原则

### 12.1 核心哲学

- **性能优先**：每一层设计（类型系统、寄存器传参、QBE 后端）都以生成高效机器码为目标。栈效应声明给编译器提供关键优化信息，fib(35) 达到 C 的 1.0x。
- **栈效应承担类型区分的职责**：不引入复杂的类型推断系统，栈效应声明（`d:d`）同时承担函数签名和类型标记的双重职责。
- **极简符号集**：14 个单符号覆盖算术、比较、控制流、内存、打印、注释、字符串。保留括号等符号给未来关键功能。
- **空格作为语法边界**：`.` 的双重语义（打印 vs 成员访问）由空格规则在词法层决定，不做运行时上下文推断。这是延续下来的核心设计。

### 12.2 类型系统设计

- **数字类型由小数点决定**：`35` = 整数 l，`35.0` = 浮点 d。无需后缀，语法层面一目了然。
- **三个类型足够**：`l`（64位整数）、`d`（64位浮点）、`p`（指针）。复杂数据类型通过结构体组合（远期）和 FFI 扩展。
- **类型化操作**：算术和比较根据操作数类型自动选择语义，类型不匹配编译期报错。0 运行时开销。

### 12.3 栈与内存

- **栈是编译期概念**：编译器内部用虚栈追踪类型和值，寄存器传参消除运行时全局栈瓶颈。
- **指针就是真实地址**：不再编码为 double 偏移量，`&` `#` 直接生成 `loadl`/`loadd`/`storel`/`stored`。
- **字符串是 p+l 双值**：指针即数据头（FFI 零转换），长度在栈上 O(1) 访问。null 结尾保证 C 兼容性。

### 12.4 模块与封装

- **命名空间即封装**：不设显式导出关键字，全部导出但通过模块名前缀隔离。简单、可预测。
- **命名导入**：`:use name 'path'` 取代扁平 `:use`，避免大型项目中的命名冲突。
- **行首规则**：`:use` 和 `:bind` 必须在行首起始、独占一行，保持源码结构清晰。

### 12.5 标准库策略

- **核心保持精简**：语言核心只包含原语和基本类型。复杂功能通过标准库（Zona 实现）和 FFI（C 库）扩展。
- **文件 IO、线性内存、调试原语**：已有成熟设计，按优先级逐步补齐（见 §7.7、§7.8）。
- **结构体**：远期实现，设计方向已确定（见 §十）。

### 12.6 兼容性

这是一次**不兼容的重设计**。主要破坏性变更：

| 特性 | 旧版 | 新版 |
|------|------|------|
| 数字类型 | 统一 double | 整数/浮点分离（由小数点区分）|
| 栈效应声明 | 无 | 强制 |
| 函数参数 | 全局栈传递 | 寄存器传递 |
| 字面量 `35` | double 35.0 | 整数 35 |
| 指针 | double 编码偏移量 | 真实地址 |
| 模块 | 扁平 `:use` | 命名导入 + 成员访问 |

新旧版本共存于同一仓库，两个编译器代码独立。
