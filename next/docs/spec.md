# Zona V2 语言规范

> 最终版。基于实际实现的编译器 (`next/src/zonac.c`)。
> 设计哲学：性能优先，栈效应承担类型区分的职责。

## 一、词法

- 空白分隔词元
- `_` 至行尾为注释

### 1.1 符号

```
@ ; : ? ! $ ~ + - * / % ^ > < = . _ ' & #
```

| 符号 | 语义 |
|------|------|
| `@` | 开始字定义 |
| `;` | 结束字定义 |
| `:` | 栈效应分隔符 / 原语前缀 |
| `?` | 条件执行（消费栈顶 l，非零为真） |
| `!` | else 分支标记 |
| `$` | 显式返回 |
| `~` | 跳回当前字头部（循环） |
| `+ - * / % ^` | 算术（类型化） |
| `> < =` | 比较（类型化，返回 l） |
| `&` | 内存读 `( p -- l|d )` |
| `#` | 内存写 `( l|d p -- )` |
| `.` | 打印（有空格）/ 成员访问（无空格） |
| `_` | 行注释 |
| `'` | 字符串定界符 |

**保留未分配**：`( ) [ ] { } , | " \``

### 1.2 字面量

**整数**（l 类型）：不含小数点。`35` `0` `-5`

**浮点**（d 类型）：含小数点。`35.0` `0.0` `-5.0`。QBE 要求浮点常量格式 `d_X.0`，编译器内部用 `%g` 输出保证正确精度。

**字符串**：单引号包裹。`'hello'`。压入 `p l` 两个值——data section 的 C 字符串指针 (null 结尾，保证 FFI 兼容) + 编译时长度。转义支持 `\n` `\t` `\\` `\'`。

### 1.3 词元类型

| 类型 | 例子 | TokenType |
|------|------|-----------|
| 数字 | `35` `35.0` | T_NUM |
| 字符串 | `'hello'` | T_STR |
| 符号 | `+` `?` `@` `:` | T_SYM |
| 系统原语 | `:dup` `:type` | T_PRIM |
| 用户字 | `fib` `abs` | T_WORD |
| 成员访问 | `math.abs` | T_MEMBER |

### 1.4 栈效应 . 词法歧义

`:` 根据上下文区分：`d:d` 中间有冒号 → 签名分隔符；`:dup` 冒号开头跟词 → 原语前缀。tokenizer 将独立 `:` 作为 T_SYM，`:xxx` 作为 T_PRIM。

## 二、类型系统

栈效应子集（8 字节栈槽）：`l` `d` `p`

| 类型 | 含义 | 字面量 |
|------|------|--------|
| `l` | 64位整数 | `35` |
| `d` | 64位浮点（IEEE 754） | `35.0` |
| `p` | 指针 / 字符串地址 | `'hello'` `:alloc` |

**类型化操作**：算术和比较根据操作数类型选择语义。`3 5 +` 整数加法，`3.0 5.0 +` 浮点加法，类型不匹配编译错误。比较始终返回 `l`（0 或 1）。

## 三、栈效应声明

每个字定义必须包含栈效应，格式为 `输入类型:输出类型`，紧跟字名之后。

```
@ 名称 输入类型:输出类型 body ;
```

- `:` 前为输入类型序列，后为输出类型序列
- 每个字母对应栈上一个值，从左到右对应栈从底到顶
- 只使用 `l d p` 三个类型标记
- 空输入/空输出省略字母

```
@ fib d:d ... ;       _ 1 个 d 入，1 个 d 出
@ add ll:l + ;        _ 2 个 l 入，1 个 l 出
@ greet : 'hi' :type ; _ 0 入，0 出
```

**重定义**：同名字覆盖先定义，编译器用最新定义。

## 四、控制流

### 4.1 条件 `?`

消费栈顶 l 类型值。每个分支只执行**一个词元**。

```
条件 ? 真词元                _ 单分支
条件 ? 真词元 ! 假词元        _ 双分支
```

`!` 标记 else 分支。分支汇合时栈类型布局必须一致，使用 `$` 返回的分支豁免汇合约束。

### 4.2 返回 `$`

立即退出当前字。函数返回结果在虚栈上，由编译器自动生成 `ret` 指令。

### 4.3 循环 `~`

跳回当前字头部。`~` 处栈类型必须与字入口处（消费输入参数后）一致，否则编译错误。

## 五、字符串表示

`'hello'` 压入 `p l` 两个值：

- `p`：data section 中 C 字符串的指针（null 结尾，保证 FFI 兼容性）。
- `l`：编译时已知的字符串长度。

`:type` 签名 `pl:`，用 `printf("%.*s", len, ptr)` 输出，不依赖 `\0`。FFI `:bind` 传 `char*` 参数时，字符串指针直接传寄存器，零开销。

**为何后置长度而非前置？** 指针即数据头，传给 C 函数时不需要偏移。长度在栈上独立存在，O(1) 访问。字符串可含 `\0`。

**为何 null 结尾？** 保证与所有 C 函数（`fopen`、`puts` 等）的兼容性。一个字节代价，FFI 边界零转换开销。

## 六、内存读写

`&` 读内存，`#` 写内存。固定 8 字节宽度。

| 符号 | 签名 | 语义 |
|------|------|------|
| `&` | `p:l` 或 `p:d` | 读 8 字节 |
| `#` | `lp:` 或 `dp:` | 写 8 字节 |

编译器生成 QBE `loadl`/`loadd`/`storel`/`stored`。

**为何不做引用封装？** 引用模型（`{addr, value}` 结构体）虽与字符串模型统一，但解释器有堆分配开销，且对调试、逐字节操作不透明。高频的 `&` `#` 应保持直接和零开销。引用能力可在标准库层实现。

**窄宽度（1/2/4字节）读写**：留给未来的结构体字段访问。`:struct` 声明中字段类型可为 `b h w`，编译器自动生成正确宽度的 load/store。

## 七、原语

系统原语以 `:` 前缀标识，与用户字词法层面不冲突。

### 7.1 栈操作

| 原语 | 签名 | 语义 |
|------|------|------|
| `:dup` | `x:xx` | 复制栈顶 |
| `:drop` | `x:` | 丢弃栈顶 |
| `:swap` | `xy:yx` | 交换栈顶两个 |
| `:over` | `xy:xyx` | 复制次栈顶 |
| `:rot` | `xyz:yzx` | 旋转栈顶三个 |

### 7.2 I/O

| 原语 | 签名 | 语义 |
|------|------|------|
| `:print` | `l:` 或 `d:` | 打印栈顶并消耗 |
| `.` | `l:` 或 `d:` | 同上（有空格时） |
| `:type` | `pl:` | 输出字符串（`printf("%.*s")`） |
| `:emit` | `l:` | 输出字符 |
| `:key` | `:l` | 读入字符 |

### 7.3 内存

| 原语 | 签名 | 语义 |
|------|------|------|
| `:alloc` | `l:p` | `malloc(n)`，返回真实指针 |
| `:free` | `p:` | `free(ptr)` |

### 7.4 系统

| 原语 | 签名 | 语义 |
|------|------|------|
| `:time` | `:l` | `time(NULL)` |
| `:rand` | `l:l` | `rand() % n` |
| `:exit` | `l:` | `exit(code)` |
| `:argc` | `:l` | 命令行参数个数 |
| `:argv` | `l:p` | 第 n 个参数指针（待实现） |

### 7.5 编译时原语

| 原语 | 语义 |
|------|------|
| `:use` | 引入文件（待实现） |

## 八、编译器架构

**QBE 后端**。复用 V1 `zonac.c` 策略，关键改进：栈效应签名 → 带类型参数的 QBE 函数 → 寄存器传参。

- `@ fib d:d` → `function d $zona_fib(d %n)` — 参数走寄存器，返回值走寄存器
- 函数内部虚栈追踪（`vstack[]`）+ 控制流边界 `vsync()`
- 所有原语直接从虚栈取操作数、推结果，不碰运行时栈
- 运行时 `$stack` / `$sp` 仅用于 `~` 循环和 `emit_pop_typed` 调用边界

**性能 (fib(35)，Apple M 系列)**：

| | 时间 | vs C |
|------|------|------|
| V2 编译器 | **0.04s** | **~1.0x** |
| V1 编译器 | 0.106s | 2.5x |
| C (-O2) | 0.043s | 1x |

## 九、模块系统

**语法**：`:use name 'path'` 命名导入。成员访问 `name.word`（解析为 T_MEMBER token）。去重基于绝对路径。路径相对当前文件所在目录解析。全部导出，命名空间即封装。别名通过普通字定义实现：`@ add3 ll:l math.add ;`。
