# Zona V2 — 设计决策记录（最终版）

> 所有决策已锁定。直接做编译器，不做解释器。
> 设计哲学：**性能优先**。一切决策以编译器能生成最优代码为准。

---

## 决策 1：字符串表示 ✅

**决定：C 字符串。p 类型，null 结尾。**

- `'hello'` 压入一个 `p`，指向 data section 的 `"hello\0"`
- `:type` 签名 `p:`，内部调用 `puts()`
- 编译器：data section 静态存储，`.string` 或 `.byte` 指令
- FFI 调用 C 函数天然一个 `x0` 传参
- 引用模型（`{len, data}`）放弃——透明性和简单性优先。
  引用操作能力可通过标准库字在用户层实现。

---

## 决策 2：内存读写 ✅

**决定：直接内存操作。`&` 读，`#` 写。不做引用封装。**

- `&` 签名 `p:l`（或 `p:d`），读 8 字节
- `#` 签名 `lp:`（或 `dp:`），写 8 字节
- 编译器生成 `loadl`/`loadd`/`storel`/`stored`
- 不封装成引用结构体——保持透明和零开销
- 引用模型（`{addr, value}`）放弃。高层次的封装留给标准库

---

## 决策 3：编译器架构 ✅

**决定：QBE 后端。利用栈效应签名生成寄存器传参的函数。**

- `@ fib d:d` → `function d $zona_fib(d %n)` — 参数走寄存器，返回值走寄存器
- 利用 QBE 的寄存器分配器
- 虚栈追踪 + vsync 策略（和 V1 zonac.c 一样）
- 目标：fib(35) 达到 C 的 1.0-1.3x

---

## 决策 4：符号表 ✅

| 符号 | 语义 |
|------|------|
| `@` `;` | 字定义 |
| `:` | 栈效应分隔符 / 原语前缀 |
| `?` `!` `$` `~` | 控制流 |
| `+` `-` `*` `/` `%` `^` | 算术（类型化） |
| `>` `<` `=` | 比较（类型化，返回 l） |
| `&` `#` | 内存引用操作 |
| `.` | 打印（有空格）/ 成员访问（无空格） |
| `_` `'` | 注释 / 字符串定界符 |

---

## 决策 5：开发顺序 ✅

```
阶段 1: QBE 编译器 — tokenizer + 字典 + 栈效应验证
阶段 2: 算术/比较 IR 生成 + 控制流
阶段 3: 引用模型 + 字符串
阶段 4: FFI (:bind)
阶段 5: 内联 + 性能测试 vs V1
阶段 6: 标准库
阶段 7: 模块系统
```

## 原语速查

| `:drop` | `x:` | 丢弃栈顶 |
| `:swap` | `xy:yx` | 交换栈顶两个 |
| `:over` | `xy:xyx` | 复制次栈顶 |
| `:rot` | `xyz:yzx` | 旋转栈顶三个 |

| `:key` | `:l` | 读入字符 |

| `:free` | `p:` | 释放内存 |

| `:rand` | `l:l` | 随机数 |
