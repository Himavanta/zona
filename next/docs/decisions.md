# Zona V2 — 设计决策记录（全部已定）

> 所有决策已锁定。本文档是开发的唯一设计依据。
> 设计哲学：**性能优先**。栈效应签名承担类型区分的职责，不靠原语名字。

---

## 决策 1：字符串表示 ✅

**决定：单指针（p 类型）**。

- `'hello'` 压入一个 `p` 值，指向 `{int64_t len, char data[]}` 结构体
- `:type` 签名 `p:`，通过 `*(int64_t*)(ptr-8)` 读长度
- 解释器：静态预分配（B2），零运行时分配
- 编译器：data section 静态存储，单寄存器传参，FFI 调用 C 函数一个 `x0` 搞定

---

## 决策 2：内存读写 ✅

**决定：保留 `&` `#` 单符号。删除 `:peek` `:poke`。**

- `&` 读内存 `( p -- l )` 或 `( p -- d )`，签名区分 `l`/`d`
- `#` 写内存 `( l p -- )` 或 `( d p -- )`，签名区分 `l`/`d`
- 固定 8 字节宽度。窄宽度（1/2/4 字节）交给未来的结构体字段访问
- 高频操作用最短语法。`:peek` `:poke` 是 V1 思维——靠名字区分类型，V2 签名已解决此问题

---

## 决策 3：调试原语 ✅

**决定：纳入 `:stack`，`:trace` 按需。**

- `:stack` 签名 `:`，打印栈内所有值，含类型标记。立即实现
- `:trace` 签名 `l:`，非0开启单步追踪，0关闭。等需要时实现
- 不加 `:break`——`:stack` + `:trace` 已足够

---

## 决策 4：局部变量 ✅

**决定：语法已设计（`|name|` 见 redesign.md §2.9），暂不实现。**

- `|` 符号预留
- 语法：`@ dist ( d d d d -- d ) |dx| x2 x1 - |dy| y2 y1 - ...`
- 实现时机：ARM64 编译器稳定后，如果栈重排确实成为可读性或性能瓶颈
- `&` `#` 与局部变量无关——它们是内存读写，局部变量用 `|name|` 声明、普通词名引用

---

## 决策 5：错误处理 ✅

**决定：远期，简单错误返回先行。**

- 现阶段：C 惯例——返回 -1 表示错误，调用方手动检查
- 未来：`( )` 预留给 try-catch 风格错误处理
- `[ ]` `{ }` 继续保留，不分配

## 决策 8：编译器架构 ✅

**决定：QBE 后端。V1 zonac.c 已验证此路线可行。**

- 复用 V1 的 QBE IR 生成策略
- 栈效应签名让编译器知道函数参数数量和类型 → 生成带参数的 QBE 函数 → 寄存器传参
- 消除 V1 的"全局数组栈"瓶颈——不再是 `function $zona_fib()` 零参数全局栈，而是 `function d $zona_fib(d %n)`
- 期望 fib(35) 从 V1 的 0.106s (C 的 2.5x) 降到接近 C

**解释器先行**：先用解释器验证语言设计，编译器在解释器稳定后开始。

## 决策 9：开发顺序 ✅

**决定：解释器先行，编译器在后。**

```
阶段 1: 完善解释器（字符串、&/#、调试/内省原语、REPL）
阶段 2: QBE 编译器 — 算术、控制流、函数调用
阶段 3: FFI 支持（:bind 按 QBE 类型生成调用代码）

阶段 6: 模块系统
阶段 7: 结构体
远期: 错误处理、独立编译、局部变量、并发
```

## 决策 10：语言级调试与内省机制 ✅

大型项目维护需要语言自身提供的调试和内省能力。这些不是外部工具，而是语言内置原语，在解释器和编译器中都可用。

### 核心调试原语

| 原语 | 签名 | 语义 |
|------|------|------|
| `:stack` | `:` | 打印栈内所有值，含类型标记和位置。不消耗 |
| `:trace` | `l:` | 非0开启步进追踪（每步打印操作+栈状态），0关闭 |
| `:depth` | `:l` | 压入当前栈深度（元素个数） |
| `:reset` | `:` | 清空数据栈和返回栈（REPL 中用于错误恢复） |

### 内省原语

| 原语 | 签名 | 语义 |
|------|------|------|
| `:words` | `:` | 列出所有已定义字，含栈效应签名 |
| `:see` | `p:` | 展示指定字的定义体（源码级反编译）`'wordname' :see` |
| `:locate` | `p:` | 展示字定义所在的文件和行号 |

### 运行时检查

| 原语 | 签名 | 语义 |
|------|------|------|
| `:assert` | `l:` | 栈顶为0则报错退出，打印调用栈 |

### 错误信息质量

- 所有错误必须包含**文件名 + 行号**
- 运行时错误（栈溢出、类型不匹配）打印**调用栈回溯**：当前字 ← 调用者 ← 调用者的调用者，每层带文件名和行号
- 栈效应验证失败时指出**声明 vs 实际的差异**

### 实现优先级

阶段 1 立即实现：`:stack` `:depth` `:words` `:see` `:assert` + 错误信息含文件名 + 调用栈回溯
阶段 1 按需实现：`:trace` `:reset`

## 类型系统速查

栈效应子集（8 字节栈槽）：`l` `d` `p`

| 类型 | 含义 | 字面量示例 |
|------|------|-----------|
| `l` | 64位整数 | `35` `0` `-5` |
| `d` | 64位浮点 | `35.0` `0.0` `-5.0` |
| `p` | 指针/字符串 | `'hello'` `:alloc` |

## 符号表（最终）

| 符号 | 语义 |
|------|------|
| `@` | 开始字定义 |
| `;` | 结束字定义 |
| `:` | 栈效应分隔符 / 原语前缀 |

| `!` | else 分支 |
| `$` | 返回 |
| `~` | 循环 |

| `>` `<` `=` | 比较（类型化，返回 l） |
